## Overview

This repository contains all code, prompts, raw outputs, and intermediate data files needed to reproduce every result in our report. The study investigates whether markdown formatting style (prose paragraphs, bullet points, markdown headers) systematically influences LLM judge preference verdicts, independently of response content.

## Environment Setup

**Python version:** 3.10 or above recommended.

Install dependencies:

```bash
pip install -r requirements.txt
```

**API key:** All scripts read your OpenRouter API key from an environment variable. Set it before running anything:

```bash
export OPENROUTER_API_KEY=your_key_here
```

You can get an OpenRouter key at https://openrouter.ai. The two models used are:
- `openai/gpt-4o-mini` — answer generation, reformatting, and judge
- `anthropic/claude-haiku-4` — judge

All API calls use `temperature=0` for reproducibility. The swap-resolution random seed is fixed at `42`.

## Reproducing the Results

The pipeline consists of five steps. **Steps 1–4 involve API calls and cost approximately $6 in total.** To skip them, use the pre-generated files in `data/` and jump directly to Step 5.

### Step 1 — Generate questions

```bash
python Prepare_Q.py
```

Reads from: HuggingFace datasets (`truthful_qa`, `commonsense_qa`), plus hardcoded procedural questions.
Writes to: `data/questions_60.csv`

### Step 2 — Generate prose answers

```bash
python Prepare_prose_A.py
```

Reads from: `data/questions_60.csv`
Writes to: `data/answers_prose.csv`

Supports resume: if interrupted, re-running skips already-completed rows.

### Step 3 — Reformat into bullet and header variants

```bash
python Prepare_Reformatting_A.py
```

Reads from: `data/answers_prose.csv`
Writes to: `data/answers_all_180.csv`

Supports resume.

### Step 4 — Run LLM judgements

```bash
python Judge.py
```

Reads from: `data/answers_all_180.csv`
Writes to: `data/judgements.csv`

Runs all 720 API calls (60 questions × 3 pairs × 2 positions × 2 judges) with swap protocol. Supports resume.

### Step 5 — Analyse results

```bash
python analyze.py
```

Reads from: `data/judgements.csv`
Writes to: `analysis_output/` (all figures and tables)

This step requires no API calls and reproduces every figure and table in the report.
